QA Manual Testing – Shopee

1. Deskripsi
Project ini berisi hasil pengujian manual pada website e-commerce (studi kasus Shopee).
Pengujian dilakukan untuk memastikan fitur utama berjalan dengan baik serta mengidentifikasi bug dan masalah pada sistem.

2. Tujuan
* Memastikan fitur utama berjalan sesuai fungsinya
* Mengidentifikasi bug dan masalah sistem
* Menganalisis pengalaman pengguna (User Experience)

3. Ruang Lingkup Pengujian
Fitur yang diuji meliputi:
* Login
* Register
* Pencarian Produk
* Checkout

4. Tools yang Digunakan
* Google Chrome
* Microsoft Excel
* Manual Testing

5. Struktur Project

QA-Manual-Testing-Shopee/
- Test-Plan-Shopee.pdf
- Test-Case-Shopee.xlsx
- Bug-Report-Shopee.pdf
- Screenshot/
- README.md
  
6. Hasil Pengujian
Dari hasil pengujian ditemukan beberapa masalah, di antaranya:
* Masalah pada proses login dan registrasi
* Kebingungan pengguna terkait metode login
* Potensi issue pada keamanan sistem
* Error pada fitur hapus akun

7. Contoh Temuan Bug
* Login otomatis tanpa verifikasi setelah registrasi ulang akun
* Server error saat menghapus akun kedua kali
* Metode login tidak jelas setelah registrasi

8. Screenshot
Screenshot hasil pengujian dapat dilihat pada folder **Screenshot**.

9. Kesimpulan
Pengujian menunjukkan bahwa masih terdapat beberapa bug dan masalah pada sistem yang dapat memengaruhi pengalaman pengguna dan keamanan akun.

10. Catatan
Project ini dibuat sebagai bagian dari pembelajaran QA Manual Testing dan portofolio untuk melamar posisi QA Intern.
